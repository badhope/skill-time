# 🎲 Random-Skill 架构设计

> **世界的底层是随机的，大脑的底层也是随机的**

## ✨ 新范式 vs 旧思维

| 维度 | 旧的 Python 大脑架构 | 新的扣子 Skill 范式 |
|------|---------------------|-------------------|
| **核心思想** | 在代码里模拟一个大脑 | 用 Markdown 写人类思维操作手册 |
| **加载方式** | 全部加载，20000 行代码硬塞进上下文 | 渐进式披露：先元数据，再流程，最后按需加载资源 |
| **模块化** | 假模块化，文件拆了但强耦合 | 真模块化，每个 Skill 独立目录，独立生命周期 |
| **随机性** | 生成几个随机参数，流程还是固定的 | **过程随机，结果收敛**，路径选择随机，中途还能变道 |
| **技术栈** | 100% Python，面向对象，设计模式满天飞 | 80% Markdown（人类经验），20% 脚本（辅助工具） |
| **可组合性** | 大脑是一个整体，拆不开 | Skill 像乐高，可以自由组装、搭配、插拔 |

## 🏗️ 三层架构设计

### 第一层：元数据层（Metadata Layer）

**位置**：每个 SKILL.md 开头的 YAML Front Matter  
**Token 消耗**：每个 Skill ~50 tokens  
**始终加载**：是

```yaml
---
name: random-thinking-skill
version: 1.0.0
description: 当用户需要创意灵感、思维跳跃、随机决策时使用
icon: 🎲
tags: [创造力, 随机思维, 灵感, 直觉]
---
```

**作用**：Agent 启动时只看这个，用来做意图识别和技能匹配。

---

### 第二层：指令层（Logic Layer）

**位置**：SKILL.md 的主体内容  
**Token 消耗**：500 - 5000 tokens  
**加载时机**：用户请求匹配到技能时才加载

包含：
- 技能的核心理念和原则
- 详细的执行流程（SOP）
- 分支路径和选择机制
- 输出格式和质量要求
- 禁止事项和边界条件

**核心价值**：把人类专家的思考过程写成标准操作流程，AI 照着做就行，不用自己重新发明。

---

### 第三层：资源层（Resource Layer）

**位置**：scripts/、references/、assets/ 目录  
**加载时机**：SKILL.md 明确指示时才读取

包含：
```
skill/
├── scripts/            # 需要计算的逻辑用 JS
│   └── random_picker.js
├── references/         # 参考资料和数据
│   ├── facts_database.json
│   └── creativity_techniques.md
└── assets/             # 静态资源
    └── templates.md
```

**特点**：大小无限制，按需加载，不浪费上下文。

## 🎲 随机性核心设计

### 随机性不是点缀，是灵魂！

#### 1. 路径选择随机
> 解决同一个问题，每次走不同的路

```
扔 D4 骰子选路线：
  1 → 🎨 创意发散路线
  2 → 🔍 分析拆解路线
  3 → 💭 走神联想路线
  4 → 🎲 纯粹随机路线
```

#### 2. 思维跳跃随机
> 走神不是 bug，是 feature

- 自由联想跳转 N 次
- 强制建立牵强连接
- 插入不相关的冷知识
- 故意引入随机干扰

#### 3. 中途变道随机
> 想着想着就想到别的地方去了，这才是人

```
执行到第 2 步时，扔 D20 骰子：
  掷出 1 或 20 → 立刻切换到另一条路线
  不需要理由，不需要解释
  变道本身就是价值
```

#### 4. 结果收敛稳定
> 不管过程怎么随机跳，最终都回到用户的目标

- 混乱是过程，不是目的
- 发散之后必须收敛
- 随机之后必须落地
- 跑偏了必须能拉回来

## 🧩 真正的模块化技能组合

```
skills/
├── random-thinking/          # 主技能：随机思维总入口
│   ├── SKILL.md
│   ├── scripts/
│   └── references/
├── creativity-stormer/       # 子技能：创意风暴
│   └── SKILL.md
├── mind-wanderer/            # 子技能：走神联想
│   └── SKILL.md
└── more skills coming...
```

### 组合方式
1. **独立使用**：用户直接调用某个子技能
2. **嵌套调用**：主技能执行到一半调用子技能
3. **链式调用**：Skill A 输出 → Skill B 输入
4. **投票机制**：多个技能同时执行，结果投票
